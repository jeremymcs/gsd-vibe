### Integration Gaps

**1. Parsing commands unconnected to frontend (tech debt, not blocker)**
PARS-01 (`gsd2_list_milestones`), PARS-02 (`gsd2_get_milestone`), PARS-03 (`gsd2_get_slice`), PARS-05 (`gsd2_get_roadmap_progress`) — all registered in `lib.rs` but have no TS invoke wrappers. The GSD-2 Milestones/Slices/Tasks tabs in `project.tsx` render "coming soon" placeholder content. This is explicitly deferred per Phase 2 plan key-decisions. Requirements PARS-01..05 describe the backend commands, not the UI consumers — commands are implemented and verified.

**2. HDLS-04: Log continuity on navigation (tech debt)**
`bufferRef.current` resets to `''` on `Gsd2HeadlessTab` unmount. Lines received while the listener was not active are permanently lost. The session itself continues running and recovers session state on remount. Severity: UX degradation, not functional failure.

**3. WORK-05 / VIZ-04: No reactive file-change invalidation (tech debt)**
`use-gsd-file-watcher.ts` only invalidates `gsd2Health` on `gsd2:file-changed` events. Worktrees and Visualizer queries rely only on 30s polling intervals. Not a requirement failure — neither WORK-05 nor VIZ-04 specify reactive updates.
